# Continuity

ANN's original roadmap is complete. Sessions 4-13 shipped the production
workflow, URL quality improvements, deterministic replay, optional outlets,
weekly retrospectives, dashboard auto-refresh, provider/model flexibility,
dashboard retrospective display, optional retrospective re-ranking, and
release hardening for v0.2.0.

## Current State

- Core pipeline: fetch candidates -> provider index-only selection -> render
  `headlines-YYYY-MM-DD.md` -> Streamlit dashboard.
- Automation: GitHub Actions runs the daily digest on schedule or manual
  dispatch and commits generated digest files safely.
- Quality: AP Google News links resolve best-effort to canonical publisher URLs;
  failures fall back to the original link.
- Replay: candidate snapshots support offline deterministic reruns.
- Coverage: optional `GUARDIAN`, `BBC`, and `NPR` outlets are available through
  `ANN_EXTRA_OUTLETS`.
- Retrospective: `ann.py retro` builds a deterministic weekly digest from prior
  daily digests, with optional provider-assisted index-only re-ranking via
  `--rerank-model`.
- Dashboard: daily digest and weekly retrospective tabs auto-check for new or
  overwritten generated files every 30 seconds.
- Providers: Anthropic, OpenAI, and Google Gemini can be selected through
  env/config or CLI flags while preserving index-only selection and env-only key
  handling (`GEMINI_API_KEY` / `GOOGLE_API_KEY`, default `gemini-2.5-flash`). The
  daily-digest workflow validates all three providers and injects each provider's
  secret; DEVOPS.md and DEVELOPMENT.md document the Gemini path.
- Release state: `pyproject.toml` is bumped to `0.4.0` (security review + production
  container), the changelog has a dated `0.4.0` section, Dev-Ops documents
  production setup and release steps, and blank GitHub Actions model variables
  fall back to defaults.
- Digest location: `ANN_DIGEST_DIR` (default: repo root) controls where digests
  and retrospectives are written and read. The container sets it to `/data`.
- Gemini reliability: `gemini-2.5-flash` thinking is disabled
  (`thinking_config.thinking_budget = 0`) for index-only selection so reasoning
  tokens cannot exhaust `max_output_tokens` and return empty text.

## Next Phase

No next phase is selected. ANN is production-ready: the security review is
complete, the container is hardened and verified, and CI gates lint, tests,
dependency audit, and container invariants. See open ideas below.

## Invariants

- Preserve the no-fabrication guarantee: the model selects candidates by index
  only; rendered titles and links come from fetched candidates.
- Model provider flexibility must not persist secrets; prefer env vars,
  deployment secrets, or in-memory UI state.
- Keep tests pure: no live-network or live-model unit tests.
- Never commit `.env`, secrets, cache snapshots, or generated bytecode.
- Never interpolate untrusted values into a `<script>` block with plain
  `json.dumps`; use `_script_json`. A literal `</script>` in a feed title closes
  the tag no matter what JS string it sits inside.
- The container image stays immutable: no source bind-mount over `/app`, no
  digests baked into layers, no dev/test tooling installed.
- Update `CHANGELOG.md`, `memory/ann_status.md`, and this file whenever a new
  project phase ships.

## Open Ideas

- Consider additional providers only behind the same normalized index-only
  interface and env/in-memory key boundaries.
- Pin GitHub Actions to commit SHAs rather than mutable major tags.
- Publish the image to a registry (GHCR) on tagged releases.

## Security Review (completed — Session 14, v0.4.0)

The dedicated security evaluation is done. Findings and resolutions:

- **Stored XSS in the dashboard (found and fixed).** `_dashboard_component`
  interpolated `json.dumps(headlines)` into a `<script>` block. A feed title
  containing a literal `</script>` closes the script element regardless of
  JavaScript string context, and `json.dumps` escapes neither `<` nor `/`. The
  prior scan's conclusion that "the injection surface is handled" was wrong: the
  `textContent` writes guard a different sink. All script-context values now go
  through `_script_json` (escapes `<`, `>`, `&`).
- **SSRF in `resolve.py` (assessed, hardened).** `_resolve` was already gated by
  `_is_google_news_url`, so arbitrary hosts were never fetched. Tightened anyway:
  the gate now requires scheme `https` *and* host `news.google.com`, and the
  resolved publisher URL is scheme-validated before adoption.
- **feedparser XXE (assessed, safe).** `feedparser==6.0.12` does not resolve
  external entities; a `<!ENTITY xxe SYSTEM "file:///etc/passwd">` payload
  yields the literal `&xxe;`. Locked in by test.
- **Supply chain (resolved).** `requirements.txt` is pinned to exact versions;
  dev tooling split into `requirements-dev.txt`; CI runs `pip-audit` (clean as
  of 2026-07-10). GitHub Actions still reference mutable major tags — pinning
  those to SHAs remains an open, low-priority idea.
- **Regression tests added.** `tests/test_security.py` covers script-context
  escaping, both URL scheme allowlists, markdown escaping, the resolver host
  allowlist, and the XXE posture.

## Container

The image is a deployable artifact, not a dev convenience:

- Non-root (uid `10001`), runtime dependencies only (no `pytest`/`ruff`).
- Digests are runtime state on a `/data` volume (`ANN_DIGEST_DIR`), never baked
  into image layers and never served from a bind-mounted source tree.
- Compose applies `read_only`, `cap_drop: ALL`, `no-new-privileges`, `mem_limit`,
  and `pids_limit`. CI asserts non-root and the absence of test tooling.
- Verified 2026-07-10: builds, runs healthy under a read-only rootfs, `/app` is
  read-only, `/data` is writable, and the dashboard parses 20 headlines from the
  mounted volume.
